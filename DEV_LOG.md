# 멸망했는데 바쁨 — 개발 로그

> 아포칼립스 배경 보드게임의 웹 프로토타입

---

## 게임 디자인 의도

- **코어 경험**: 팀플의 어려움 — 극한의 상황에서 서로 다른 특성을 가진 생존자들끼리 불합리함을 견디며 가능한 유용한 파티를 만들어 탈출하는 경험
- **탐험 경험**: 무엇이 있는지 모르는 세상을 조금씩 발견하며 필요한 물자를 찾아가는 경험. 카드는 기본적으로 뒤집힌 상태이며 자원을 소모해 확인
- **인터랙션**: 내 파티 내부 협력 + 타 파티 방해/협력 + 생존자의 자기 의지(이탈, 반란 등)
- **구조**: 경쟁 멀티플레이어, 각 플레이어는 지도자 1명 + 파티(생존자 라인업)

---

## 프로젝트 구조

```
/src
  /components
    App.jsx              — 화면 라우팅 (setup / game)
    SetupScreen.jsx      — 플레이어 수, 지도자, 봇 설정
    GameBoard.jsx        — 메인 게임 레이아웃
    EventZone.jsx        — 이벤트 덱 + 슬롯 3개 (가로 배치)
    PlayerPanel.jsx      — 플레이어 정보 + 파티 슬롯
    PartySlot.jsx        — 파티 라인업 + 상성 점수 표시
    MercenaryPool.jsx    — 용병소 (카드 공개/영입)
  /data
    leaders.js           — 지도자 5명 정의
    survivors.js         — 생존자 카드 + 상성 매트릭스
    events.js            — 이벤트 카드 (개인/전체/재난)
    rules.js             — 게임 상수
  /logic
    scoring.js           — 상성 점수 계산, 주사위
    bot.js               — 봇 AI 로직
  /store
    gameStore.js         — Zustand 전역 상태
/scripts
  importCards.js         — CSV → JS 변환 스크립트
```

## 카드 종류

### 지도자 (5명)
| 이름 | 속성 | 고유 스킬 | 특수 승리 조건 |
|------|------|-----------|---------------|
| 샤이 | 겁쟁이 | 울기 (자원 획득) | 자원 30개 보유 |
| 비 | 돌격대장 | 자신감있는 좀비행동 (생존자 좀비화) | 좀비 5명 보유 |
| HAL-4935 | 군기반장 | 행복수치 계산 (카드 선택권) | 파티 5명 양수 상성 |
| 덕이 | 4차원 | 오리의 기적 (생존자 흡수) | 파티 전원 4차원 |
| 소염 더 고트 | 분위기메이커 | 콘서트 (상성 보너스 ±) | 분위기메이커 3명 보유 |

### 생존자 속성 및 상성 매트릭스
```
         군기반장  분위기  돌격대장  4차원  겁쟁이
군기반장     -2       0      -1     -1      0
분위기메이커   0      +1      +1      0     +1
돌격대장     -1      +1      -1      0      0
4차원       -1       0       0      0     -1
겁쟁이        0      +1       0     -1      0
평범          0       0       0      0      0  (+전원 평범 보너스 +3)
```

### 이벤트 카드 (3종)
- **개인형**: 현재 턴 플레이어가 주사위+파티 스탯으로 해결 → 자원 보상
- **전체형**: 공개 즉시 모든 플레이어에게 동시 발동
- **재난형**: 공개 즉시 강한 부정적 효과 전체 적용

## 핵심 게임 루프

```
게임 시작
  └─ 지도자 선택 → 초기 자원 지급 → 이벤트 슬롯 3개 뒤집어 배치

매 턴 (현재 플레이어)
  ├─ 이벤트 슬롯 확인 (통조림 1개 소모) → 개인/전체/재난 처리
  ├─ 용병소 카드 확인 (병뚜껑 1개 소모) → 영입 (병뚜껑 추가 소모)
  ├─ 파티 순서 변경 (병뚜껑 2개 소모)
  └─ 턴 종료 → 수확 (평범 생존자 × 병뚜껑 1) + 유지비 (생존자 수 × 통조림 1)

이벤트 슬롯 보충
  └─ 해결된 카드 → 해결 플레이어의 버린 더미 이동 → 전체 덱에서 새 카드 뒤집어 보충
```

## 자원 체계

| 자원 | 획득처 | 소모처 |
|------|--------|--------|
| 🪙 병뚜껑 | 이벤트 보상, 평범 생존자 매 턴 생산 | 용병소 카드 확인, 생존자 영입, 파티 순서 변경 |
| 🥫 통조림 | 이벤트 보상, 초기 지급 | 이벤트 카드 확인, 생존자 유지비 |

---

## 봇 AI (V1 — 랜덤/단순 전략)

1. 통조림이 있으면 이벤트 슬롯 확인 → 개인 이벤트 자동 해결
2. 용병소 전체 카드 확인 (비용 없이 — 봇 특권)
3. 상성 계산 후 점수를 가장 높이는 생존자 영입
4. 파티 순서 변경 시 점수가 오르는 경우만 실행

## 개발 이력

### 세션 1
- 게임 룰 구상 완료 (보드게임 레이어 제외, 카드게임 레이어 집중)
- Vite + React + Zustand + Tailwind 프로젝트 세팅
- 데이터 파일 작성 (지도자, 생존자, 이벤트, 룰 상수)
- Zustand 게임 스토어 구현
- 게임 로직 구현 (상성 계산, 이벤트 해결, 주사위, 봇)
- UI 컴포넌트 구현 (설정 화면, 게임 보드, 파티, 이벤트존, 용병소)
- CSV → JS 변환 스크립트 (`npm run import-cards`)

### 세션 2
- 용병소 카드 공개 mechanic 추가: 병뚜껑 1개 소모 → 뒤집어 확인
- 봇 버그 수정: `survivorId` → `survivorUid` 불일치 수정
- 봇 자동 실행 흐름 정상화 (이벤트 해결 → 영입 → 턴 종료)

### 세션 3
- **이벤트 존 재설계**: 단일 카드 → 슬롯 3개 + 전체 덱 (가로 배치)
- 이벤트 카드 뒤집기 mechanic: 통조림 1개 소모 → 확인
- 전체/재난 이벤트: 공개 즉시 발동
- 이벤트 해결 후 플레이어별 버린 더미 귀속 + 덱에서 새 카드 보충
- 게임 레이아웃 재편: **이벤트 존을 플레이어 사이에 배치**
- 상대 플레이어 compact 뷰 추가

### 세션 4
- 이벤트 슬롯 수 `3 → 4` 확장
- 공개된 개인 이벤트를 `넘기기` 가능하도록 변경, 넘긴 이벤트는 플레이어별 벌점 더미로 이동
- 벌점 더미의 이벤트는 장당 `-1점`으로 총점에 반영
- 새로 보충된 이벤트 카드는 다시 뒤집힌 상태로 들어오며, 비용을 지불해야 재확인 가능
- 용병소 공개 생존자는 누군가 데려가기 전까지 계속 공개 상태 유지
- 공개된 용병소 생존자를 `통조림 1개`로 교체 가능하도록 구현
- 용병소 공개 카드 수를 `3장 → 2장`으로 줄이고, 2장을 가로 배치로 재구성

### 세션 5
- 생존자 카드 설명을 파티 영역 카드 본문에 직접 노출
- 내 파티뿐 아니라 상대 파티도 동일한 확장형 카드 레이아웃으로 통일
- 용병소에서 생존자를 영입하면 해당 슬롯은 즉시 빈 슬롯으로 남고, 턴 종료 시에만 새 카드로 보충되도록 변경
- 용병소 빈 슬롯 UI 추가
- 버린 생존자 공용 더미(`discardPile`) 도입
- 턴 종료 시 통조림이 부족하면 부족 수만큼 생존자를 직접 선택해 버린 더미로 보내도록 구현
- 봇도 유지비 부족 시 낮은 가치 생존자부터 자동 폐기하도록 대응

### 세션 6
- 생존자 턴 종료 정산을 데이터 기반으로 재설계
- 기본 생존자 정산을 `통조림 -1 / 병뚜껑 +1`로 통일
- 특수 생존자는 데이터에 따라 기본 정산을 덮어쓰거나 추가 보정 적용
- `숙련된 생존자`, `티라노 생존자`, `걷는식물 생존자`, `마법소녀 생존자` 등 예외 정산 처리 반영
- 첫 개인 턴은 이벤트 단계를 생략하고 `구매 단계만 수행`하도록 턴 구조 수정
- 봇도 첫 턴에는 구매 단계만 수행하도록 로직 정리

### 세션 7
- 이벤트 보상 체계를 리팩터링하여 `점수 / 공격 / 자원 / 조우` 4개 카테고리로 분리
- 점수형 이벤트는 `scoreTokens`를 획득하고 총점에 반영
- 자원형 이벤트는 병뚜껑/통조림을 즉시 정산
- 공격형 이벤트는 타 플레이어 대상 인터랙션으로 연결
- 조우형 이벤트는 생존자 덱에서 랜덤 생존자를 획득하도록 구현
- 불이익형 개인 이벤트는 공개 즉시 자동 시작되며, 실패 시 벌점 더미로 들어가도록 규칙 추가

### 세션 8
- 액션 결과를 알려주는 중앙 팝업 연출 시스템 추가
- 공격형 이벤트 결과(생존자 제거, 탈취, 전가 등)를 팝업으로 표시
- 전체/재난 이벤트도 즉시 발동 시 팝업 요약을 표시하도록 확장
- 턴 종료 자원 정산 시 `병뚜껑 획득`, `통조림 소모`도 순차 팝업으로 표시
- 재난 연출에 화면 흔들림과 카드 균열 이펙트 추가

### 세션 9
- 이벤트 카드를 세로형 비율로 키우고 정보 순서를 `제목 → 내용 → 해결 시작/넘기기`로 재정렬
- 버튼이 카드 프레임 밖으로 튀어나가지 않도록 카드 내부 레이아웃과 여백 조정
- 중앙 레이아웃을 재편하여 이벤트 존과 용병소를 나란히 배치
- 좁은 해상도에서도 이벤트 존 옆에 용병소가 배치되도록 간격과 패널 폭 최적화

### 세션 10
- 플레이어 패널을 `좌측 지도자 정보 / 우측 생존자 슬롯` 구조로 재배치
- 좌측 영역에 지도자 이름, 현재 턴 상태, 총점, 파티 점수, 통조림, 병뚜껑 정보를 세로 밀집 배치
- 총점/파티점수, 통조림/병뚜껑 표기를 각각 한 줄로 압축
- 지도자 정보 영역 폭을 여러 차례 축소해 생존자 슬롯 공간을 최대한 확보
- 파티 생존자 카드와 빈 슬롯의 크기를 동일한 프레임으로 통일

### 현재 구현 메모
- 개인 이벤트는 뒤집힌 상태로 확인 비용을 지불해 공개
- 공개된 해로운 개인 이벤트는 일부 카드가 자동 시작되며, 실패 시 벌점으로 처리
- 해결형 이벤트 보상은 카드 카테고리에 따라 점수/자원/공격/조우로 분기
- 용병소 공개 카드는 유지되며, 영입 전까지 남아 있음
- 턴 종료 시 자원 정산과 유지비 부족 폐기, 자원 팝업 연출까지 한 흐름으로 연결됨
